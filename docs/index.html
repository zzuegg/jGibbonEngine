Capabilities

Everything you need to build your game

jGibbonEngine combines a battle-tested JVM heritage with a modern, extensible Java API — so you spend more time building worlds, less time fighting frameworks.

🎨

Modern Rendering

Physically-based rendering (PBR), HDR lighting, dynamic shadows, and a flexible material system. Target OpenGL and Vulkan backends.

⚙️

Entity-Component-System

Cache-friendly ECS architecture gives you clean, composable game logic with blazing-fast iteration over component pools.

🏔️

Scene Graph

Hierarchical scene graph for intuitive 3D world organisation. Attach, detach, and transform nodes with a clean, fluent Java API.

🎮

Input System

Declarative, rebindable input mapping. Gamepad, keyboard, mouse and touch support out of the box.

🔊

3D Audio

Spatial audio with HRTF support. Play positional sound sources that respond to the scene — right out of the box.

💡

Clean Java API

Straightforward, well-documented Java API for scene construction, shader configuration, and asset loading. No boilerplate, no friction.

🧩

Plugin Architecture

Swap or extend any subsystem through a clean plugin API. Physics, networking, and UI are all first-class citizens.

📦

Gradle / Maven Ready

Published to Maven Central. Add a single dependency and start rendering your first scene in minutes.

🌍

Cross-Platform

Build once, run everywhere the JVM runs — Windows, macOS, and Linux desktop targets supported.

Quickstart

Up and running in 3 steps

No IDE plugin required. Add the dependency, write your first scene, and press play.

1

Add the dependency

In your build.gradle.kts: dependencies { implementation("io.github.zzuegg:jGibbonEngine:0.1.0") }
2

Create your application

Extend GibbonApplication and override init(): public class MyGame extends GibbonApplication { @Override public void init() { getRootNode().attachChild(new Box(1f)); } }
3

Run it!

Start the engine from your main method: public static void main(String[] args) { new MyGame().start(); }
